# Advanced-RAG 性能基准测试报告

> 测试时间: 2026-07-06  
> 测试环境: Windows, Python 3.10, CPU 模式  
> 测试脚本: `tests/benchmark_performance.py` + `tests/bench_core_memory.py`

## 一、核心指标总览

| 指标 | 结果 | 说明 |
|------|------|------|
| Embedding 冷启动 | 15,855 ms | 首次加载 BAAI/bge-base-zh-v1.5 + 编码 |
| Embedding 预热后 | 42.7 ms | 单次编码耗时 |
| Embedding 加速比 | 370.9x | 预热 vs 冷启动 |
| 重排模型冷启动 | 8,015 ms | 加载 BAAI/bge-reranker-base + 20对重排 |
| 重排模型预热后 | 888.7 ms | 20对重排 |
| 重排加速比 | 9.0x | 预热 vs 冷启动 |
| Base Prompt 缓存加速 | 7,320.7x | 缓存命中 vs 首次读取 |
| Base Prompt 缓存平均 | 0.0564 ms | 100次平均 |
| Core Memory 读取 | 2.17 ms | MongoDB 查询，20次平均 |
| Core Memory 追加 | 10.17 ms | 写入 MongoDB |
| Core Memory 替换 | 7.06 ms | 精确子串替换 |
| Core Memory 渲染 | 2.86 ms | 生成 prompt 文本 |
| Archival Memory 插入 | 127.38 ms | 向量化 + Qdrant 写入 |
| Archival Memory 检索 | 54.80 ms | top_k=5 语义检索 |
| LLM 首 Token 延迟 | 3,730 ms | 流式 TTFB（mimo-v2.5） |
| LLM 总响应时间 | 8,807 ms | 短回答平均（3个问题） |

## 二、模型预热效果

### Embedding 模型（BAAI/bge-base-zh-v1.5）

| 操作 | 耗时 |
|------|------|
| 冷启动（加载 + 首次编码） | 15,855 ms |
| 预热后单次编码 | 42.7 ms |
| 批量编码（10条） | 72.1 ms（7.2 ms/条） |
| **加速比** | **370.9x** |

> 结论：启动预热可消除约 15.8 秒的首次请求延迟。

### CrossEncoder 重排模型（BAAI/bge-reranker-base）

| 操作 | 耗时 |
|------|------|
| 冷启动（加载 + 20对重排） | 8,015 ms |
| 预热后 20对重排 | 888.7 ms |
| **加速比** | **9.0x** |

不同候选数重排耗时：

| 候选数 | 耗时 |
|--------|------|
| 10 对 | 506.6 ms |
| 20 对 | 887.6 ms |
| 50 对 | 2,191.8 ms |
| 100 对 | 4,592.3 ms |

> 结论：启动预热可消除约 8 秒的首次重排延迟。

## 三、缓存优化效果

### Base Prompt 缓存（5分钟 TTL）

| 操作 | 耗时 |
|------|------|
| 首次调用（读 DB / 生成默认值） | 1,330.9 ms |
| 缓存命中 | 0.182 ms |
| 100次平均 | 0.0564 ms |
| **加速比** | **7,320.7x** |

> 结论：缓存后每次请求节省约 1.3 秒的 DB 查询 + 字符串拼接开销。

## 四、记忆系统性能

### Core Memory（MongoDB）

| 操作 | 耗时 | 说明 |
|------|------|------|
| 读取 | 2.17 ms | 20次平均 |
| 追加 | 10.17 ms | 含写入 |
| 替换 | 7.06 ms | 精确子串替换 |
| 渲染 prompt | 2.86 ms | 生成注入文本 |

### Archival Memory（Qdrant）

| 操作 | 耗时 | 说明 |
|------|------|------|
| 插入 | 127.38 ms | 向量化 + Qdrant 写入 |
| 检索 | 54.80 ms | top_k=5 语义检索 |

## 五、LLM 流式响应

| 指标 | 耗时 |
|------|------|
| 平均 TTFB（首 Token） | 3,730 ms |
| 平均总响应时间 | 8,807 ms |

> 说明：TTFB 受网络和模型服务影响，这是优化代码本身无法改变的外部因素。前端已通过脉冲点占位符缓解用户等待感知。

## 六、优化收益总结

| 优化项 | 优化前 | 优化后 | 收益 |
|--------|--------|--------|------|
| Embedding 预热 | 15,855 ms | 42.7 ms | 370.9x |
| 重排模型预热 | 8,015 ms | 888.7 ms | 9.0x |
| Base Prompt 缓存 | 1,330.9 ms | 0.0564 ms | 7,320.7x |
| Core Memory 读取（缓存后） | ~17 ms | 2.17 ms | 7.8x |
| Archival Memory 检索 | - | 54.80 ms | 稳定 |
