## 关键发现
- Letta 有三个 agent 版本：letta_agent.py (1983行) / v2 (1487行) / v3 (2134行)
- `message_ids` 是 AgentState 里记录当前 in-context memory 中消息 ID 的字段
- `llm_config` 和 `embedding_config` 已废弃，改用 `model` 和 `embedding` 字段
- `_step()` 是主循环核心，涉及上下文拼装、function_call 路由、message_ids 持久化三个锚点
- tool_rules 是有限状态机约束（InitToolRule / TerminalToolRule / ChildToolRule / MaxStepsPerToolRule）
- LLMClient 工厂模式：按 provider_type 创建不同 API 客户端
- _handle_ai_response 是"执行器+决策器"：处理审批、执行工具、检查规则、持久化
- Step 记录提前写入 PENDING 状态，崩溃时也有记录可排查
